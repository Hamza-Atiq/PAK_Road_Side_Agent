# RoadSide Mobile — Flutter (customer-only at v2.0)

Hand-written scaffold. Native iOS + Android shells are added by `flutter create` overlay.

## Stack

- Flutter 3.41.9 (installed at `C:\Users\HP\flutter_windows_3.41.9-stable`)
- Dart SDK 3.4+
- State: `flutter_riverpod`
- Router: `go_router`
- HTTP: `dio` (mirrors @roadside/api-client patterns)
- WebSocket: `web_socket_channel`
- Storage: `flutter_secure_storage` (JWT) + `shared_preferences`
- Location: `geolocator` + `permission_handler`
- Camera + voice: `image_picker`, `record`
- Push: `firebase_messaging`
- Maps: `flutter_map` + OpenStreetMap tiles (matches backend ORS routing for free)
- Social sign-on: `sign_in_with_apple`, `google_sign_in`, `flutter_facebook_auth`

## First-time setup (one-time, on a workstation with Flutter)

```bash
cd apps/mobile

# 1. Overlay the native iOS + Android shells onto this hand-written scaffold.
#    --org sets the bundle ID (com.roadsideagent.mobile) — locks in App Store namespace.
flutter create . --org com.roadsideagent --project-name roadside_mobile --platforms=ios,android

# 2. Install Dart deps
flutter pub get

# 3. Run on a device or simulator
flutter run
```

After `flutter create`, you'll see new `ios/`, `android/`, and `test/` directories that are required for builds. The lib/ hand-written code above is preserved.

## Configuration

| Env var (Dart side) | Where it's read | Value |
|---|---|---|
| `RS_API_BASE_URL` | `ApiClient(baseUrl: ...)` | `https://api.roadsideagent.com` |
| Firebase config | `lib/firebase_options.dart` (generated by `flutterfire configure`) | Per-environment |

Use `--dart-define=RS_API_BASE_URL=...` when running or building.

## Building for stores (Week 6 of V2_PLAN.md)

```bash
# EAS-equivalent for Flutter is just Flutter's own build commands.
flutter build ipa --release         # iOS — produces .ipa for App Store Connect
flutter build appbundle --release   # Android — produces .aab for Play Console
```

Apple Developer Program ($99/yr) + Google Play Console ($25 one-time) required.
